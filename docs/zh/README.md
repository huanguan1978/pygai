## Gai 与 PyGai：您的智能自动化AI实践平台

本文档旨在简洁明了地介绍 Gai 应用及其开源连接器 PyGai 的核心功能、快速入门方法，并重点阐述两者在实现智能定时任务时的协同工作流程。

---

### I. Gai：您的AI创作伴侣

`Gai` 是一款专为降低生成式AI使用门槛、赋能初学者掌握AI应用精髓而设计的应用。它提供简洁直观的用户界面，所有核心功能（如提示词管理、系统指令集及生成内容存储）均在用户设备本地离线运行，充分保障数据隐私与安全。`Gai` 仅在请求外部AI服务进行内容生成时联网，且不留存任何用户输入及生成结果，确保数据可控。用户可选择性注册，不注册也可体验全部核心能力。

**主要功能特色：**

*   **本地优先，保障数据隐私：** 所有用户数据、配置及生成内容均离线存储在本地设备。
*   **极简交互，专注AI创作：** 提供直观界面，简化AI生成流程，帮助用户从使用到精通。
*   **弹性注册，功能无损：** 注册为可选功能，不影响核心AI生成能力的使用。

### II. PyGai：无限扩展的AI连接器

`PyGai` 是 `Gai` 的强大开源连接器，旨在赋能用户进行二次开发与集成。它允许通过 Python 代码扩展 `Gai` 的功能，实现与外部系统的无缝对接，构建定制化的AI工作流。`PyGai` 充当 `Gai` 应用与外部服务之间的桥梁，负责任务调度、指令分发以及结果的集中处理。

**核心能力：**

*   **数据输入 (PromptIngest)：** 通过编程方式向 `Gai` 动态输入提示词（Prompt），并支持关联的系统指令、生成参数及安全等级的配置，实现条件化批量内容生产。
*   **数据输出 (ContentDelivery)：** 将 `Gai` 生成的内容作为数据源，导出至外部系统，支持用户进行下游二次加工（如内容分发至网站、发布评论等）。

### III. Gai 与 PyGai 的协同：智能定时任务

`Gai` 与 `PyGai` 的结合，能实现强大的自动化工作流，尤其在定时任务方面。`Gai` 作为AI内容生成的核心执行者，`PyGai` 则作为智能调度与数据桥梁，共同完成内容的自动化生成、处理与分发，从而提升工作效率并减少人工干预。

#### 工作流程概述：

当 `Gai` 和 `PyGai` 协同进行定时任务时，大致遵循以下步骤：

1.  **PyGai 提供指令 (指令来源)**：`PyGai` 连接器根据预设的条件（如时间、主题等），通过其API动态筛选并向 `Gai` 应用输出一个或多个待生成的提示指令（Prompt）。
2.  **Gai 生成内容 (任务定义)**：`Gai` 应用接收到来自 `PyGai` 的指令后，在本地利用AI模型生成文本或图片内容。
3.  **Gai 回传结果 (结果处理)**：`Gai` 将生成的内容回传至 `PyGai` 连接器，同时也会在本地记录任务执行日志。
4.  **PyGai 后续处理 (结果处理)**：`PyGai` 连接器接收到生成内容后，根据预设的扩展机制（`PyGaiCustomize`），进行存储、发布到外部系统（如静态网站、CMS），或触发其他定制化的业务逻辑，实现内容自动化分发。

#### 核心要素：

*   **指令来源：** 定时任务的输入指令可来源于 `Gai` 应用本地的用户提示词库，或通过 `PyGai` 的 API 动态获取。
*   **任务定义：** 用户可通过 Cron 表达式灵活配置任务的执行频率（如每小时、每日），并指定 `Gai` 需要执行的具体AI生成指令。
*   **结果处理：** `Gai` 将生成内容自动归档至本地日志，并通过 API 回传至 `PyGai` 以支持集中管理或进一步的业务处理，同时支持通过邮件通知任务状态。

#### 典型应用场景：

*   **批量内容生成：** 针对同一主题或提示词，自动化规模化生产文章、产品评论、营销文案等。
*   **多样化内容拓展：** 基于不同提示词，生成多维度、多视角的系列内容（如主题系列文章、多角度新闻报道）。
*   **内容自动化发布与分发：** 将AI生成的内容自动发布到静态网站、内容管理系统 (CMS)、社交媒体或触发邮件营销活动。
*   **数据分析与互动：** 结合 `PyGai` 扩展机制，对生成内容进行分析，或集成评论、回复等互动功能。
